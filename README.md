# 이건가…?🤔(Was it here..?)

## 🎯 목표

```
화려함 보다 근본에 집중해보자
```

미션의 의도를 정확하게 파악했는지 모르겠지만 개인적인 오픈 미션의 목표는 보기에 화려하고 큰 규모의 프로젝트를 만드는 것이 아닌 “근본에 집중하자.”이다. 지금까지는 주로 React 환경에서 개발을 진행하며 편리하게 컴포넌트 구조와 상태관리를 해왔다. 나의 경우 전공자가 아니라서 개발의 기본기가 좀 부족한데 최근들어 기본을 놓치고 있는게 아닐까 생각이 들었다. 특히, 타입스크립트를 사용한 경험은 있지만 타입스크립트의 정적 타입 시스템과 객체지향적인 구조 설계는 중요한 요소임에도 깊게 다뤄본적이 없었다.
따라서 이번에는 React나 기타 라이브러리의 도움 없이 순수 TypeScript로 게임 로직을 직접 구현하고, 클래스와 인터페이스를 활용한 객체지향 설계를 중심으로 프로젝트를 진행하고 테스트와 수치로 프로젝트를 개선해 나가는 것이 목표이다.

- **타입스크립트에 익숙해지기 위해 타입스크립트를 적극적으로 활용한다.**
  - 기존에도 타입스크립트를 사용해 본 경험은 있으나, _타입스크립트다운_ 코드 스타일을 충분히 활용하지 못했다고 느낀다.
  - 이번 목표는 컴파일 옵션 이해부터 풍부한 타입 선언 방식, 제네릭, 유틸리티 타입 등 타입스크립트 고유의 기능을 체계적으로 익히는 것이다.
- **객체지향 프로그래밍(OOP) 원칙에 기반하여 코드를 작성한다.**
  - 주로 React 중심의 개발 환경에 익숙하다 보니 클래스 기반 프로그래밍에 대한 경험이 부족하다.
  - 이번 미션에서는 클래스 문법, 상속, 캡슐화, 인터페이스 설계 등 객체지향적인 구조를 자연스럽게 활용하는 것이 목표이다.
- **React나 기타 프레임워크/라이브러리에 의존하지 않고, 순수 TypeScript 기반으로 프로젝트를 진행한다.**
  - React를 사용하면 DOM 조작이나 상태 관리가 편리하지만, 이러한 편의성 때문에 핵심 로직과 구조 설계 역량을 충분히 쌓지 못하는 경우가 많다.
  - 이번 미션에서는 UI 프레임워크의 도움 없이 순수한 로직 구현과 설계에 집중하여, 프레임워크가 바뀌더라도 흔들리지 않는 기반 역량을 기르는 것을 목표로 한다.
  - TypeScript 환경 구성에는 Vite를 활용한다.
- **단위 테스트부터 E2E 테스트까지 진행해 본다.**
  - 기존에는 단위 테스트만 경험했으며, 복잡한 테스트 작성이나 테스트 친화적인 코드 작성에는 익숙하지 않았다.
  - 이번 미션에서는 Jest로 단위 테스트를 포함해, Playwright를 활용한 E2E 테스트까지 직접 실행하고 검증해본다.

## **📜** 설계

### 기능 목록

1. **초기 화면**
   - 화면은 게임 시작 버튼과 게임 설명으로 구성된다.
     - 정답 보드에 칠해진 위치를 외우고 동일하게 클릭하여 모든 칠해진 부분을 찾는 게임.
   - 사용자가 게임 시작 버튼을 클릭하면 스테이지 1이 시작된다.
2. **게임 진행**
   - 게임은 총 스테이지 20까지 존재한다.
   - 보드는 10 × 10 크기(총 100칸)를 가진다.
   - 매 스테이지마다 랜덤한 칸들이 정답 칸으로 표시된다.
     - 스테이지마다 정답 칸 수와 위치가 다르다.
   - 정답 보드는 일정 시간(약 2초) 동안만 표시된 후 정답 표시가 사라진다.
   - 플레이어는 제한 시간(30초) 안에 정답 칸을 모두 클릭해야 한다.
   - 클릭한 칸이 정답 칸인 경우 해당 칸은 선택 상태로 고정된다.
   - 잘못된 칸 클릭 시, 이미 맞게 선택한 칸은 유지되며 오답 칸만 원래 상태로 돌아간다.
   - 오답 칸을 클릭할 때마다 기회가 1 감소한다.
3. **아이템 시스템 (추가)**
   - 게임 보드에는 확률적으로 아이템이 생성될 수 있다.
   - 아이템 칸은 정답칸 중에서 랜덤으로 등장한다.
   - 아이템 종류:
     - 시간 추가 아이템 칸: 클릭 시 제한 시간이 +2초 증가한다.
     - 기회(라이프) 추가 아이템 칸: 클릭 시 기회가 +1 증가한다.
   - 아이템 칸은 정답 표시 시간 동안에도 표시되지 않으며, 플레이 도중 우연히 찾는 형태로 동작한다.
   - 각 아이템 효과는 즉시 적용된다.
4. **기회 및 게임 오버 처리**
   - 플레이어는 총 3번의 기회를 가진다.
   - 다음의 경우 기회가 1 감소한다:
     - 정답이 아닌 칸 클릭
     - 제한 시간 내 정답 칸을 모두 찾지 못한 경우
   - 기회가 0이 되면 게임이 종료되고 결과 화면으로 이동한다.
5. **스테이지 클리어 조건**
   - 정답 칸을 모두 정확하게 클릭하면 해당 스테이지를 클리어한다.
   - “다음 스테이지” 버튼 클릭 시 다음 스테이지로 진행한다.
6. **게임 종료 및 결과 화면**
   - 다음 정보가 표시된다:
     - 최종 도달 스테이지
     - 남은 기회 수
     - 스테이지별 클리어 시간
     - 다시 하기 버튼
   - 게임 종료 조건:
     - 모든 기회를 소진한 경우
     - 스테이지 20을 모두 클리어한 경우
